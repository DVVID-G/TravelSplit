Ecosistema Cursor

Comparativa detallada entre la Inteligencia (Agent), el Contexto (Rules) y la Ejecución (Skills).

Cursor Agent

El Cerebro / Coordinador

Definición

Es la entidad de IA central que orquesta la interacción. Lee el código, planifica y decide qué herramientas usar.

Función Principal

Toma decisiones. Decide si debe consultar una Regla o invocar una Skill.

Naturaleza

Proactiva & Reactiva. Responde al usuario y busca soluciones.

COMANDO CLAVE

Cmd+L / Ctrl+L (Chat)

Project Rules

El Manual de Normas

Definición

Instrucciones de texto que definen el estilo, formato y restricciones del proyecto. Es el 'contexto' persistente.

Función Principal

Guía el comportamiento. Le dice al Agente 'CÓMO' debe escribir el código o responder.

Naturaleza

Pasiva. Son leídas por el Agente para modular su respuesta.

UBICACIÓN

.cursor/rules/*.mdc

Skills

La Caja de Herramientas

Definición

Paquetes de capacidades que pueden incluir scripts ejecutables. Permiten al agente interactuar con sistemas externos.

Función Principal

Ejecuta acciones. Le da al Agente 'habilidades' para realizar tareas (DB, API, Deploy, Testing).

Naturaleza

Activa / Ejecutable. Puede contener código (.py, .sh) que se corre realmente.

UBICACIÓN

.cursor/skills/nombre/

La Analogía Definitiva

Si estuvieras contratando a un Desarrollador Senior (El Agente)...

  • Las Rules serían el documento PDF de "Guía de Estilo" que le entregas el primer día.
  • Las Skills serían el acceso a la terminal, las claves de AWS y los scripts de base de datos que le instalas en su laptop.

Ejemplo Práctico

Rule: "Usa siempre functional components en React."
Skill: "Ejecutar script para validar la integridad de la base de datos."
